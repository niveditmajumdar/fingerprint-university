Overview

What is Fingerprint?

Fingerprint is a device intelligence platform that assigns a stable visitor ID to each browser — one that persists beyond cookies, incognito mode, VPNs, and browser resets.

🔑 Core concepts

  • Visitor ID — Stable probabilistic identifier. Survives cookie clears and incognito mode.
  • Request ID — Unique ID per identification call. Used to fetch Smart Signals from the Server API.
  • Confidence score — 0–1 value indicating certainty. Above 0.7 is high confidence.
  • Suspect Score — Weighted sum of triggered Smart Signals. Higher = more suspicious.

Bot Detection

Analyses hundreds of browser attributes to distinguish automated traffic from real users.

🤖 Bot categories

  • not_detected — Likely a human using a regular browser.
  • good — Known bots: search crawlers, monitoring tools.
  • bad — Automation tools like Selenium, Puppeteer, or Playwright.

How does this PoC work?

1
Sign In click → Fingerprint JS agent runsThe agent collects 300+ browser signals and calls the Fingerprint API, returning a visitorId and requestId. This fires on button click — not page load — so incognito mode and VPN state at authentication time are captured.
2
Visitor ID registered server-sideThe visitorId and signals are sent to POST /api/fp/identify. The server keys all counters to the visitorId, not the email address.
3
Cached in sessionStorage — no repeated API callsThe full FP result is stored in sessionStorage after identification. The videos page reads from cache, so no second Fingerprint API call is made.
4
Video count tied to visitorId, not emailLog out, switch accounts, use incognito — the server tracks views by visitorId. A different email on the same device hits the same counter.
5
Suspect score checked on tab switch + cert submitThe cached suspect score is evaluated each time you switch tabs or submit the certification test. If it exceeds the threshold (12), a step-up verification modal appears.
6
Certification attempt tracked per deviceOne free attempt per visitorId. Creating a new account doesn't reset it.
7
Smart Signals via Server APIThe ⚡ Smart signals footer link calls GET /api/smart-signals on the backend, which calls GET /events/:requestId on the Fingerprint Server API using your secret FP_API_KEY. Results are never exposed client-side.
Note on Suspect Score: The full Suspect Score is only available via the Fingerprint Server API. In this demo it is computed client-side from available signals to simulate the behaviour. In production, fetch it from GET /v4/events/:requestId on your backend.
